## 0. 흐름 한 줄

홈에서 카드를 위로 스와이프하며 탐색하다, 마음에 드는 캐릭터의 CTA를 누르면 매칭 성공 모달 → 채팅으로 진입.
데이트 탭에서는 진행 중·완료 캐릭터를 별도 그리드로 관리.

```
[홈 피드] ─ swipe up ─→ [홈 피드(다음 카드)]
   │
   └─ "대화 시작하기" tap ─→ [매칭 성공]
                                │
                                ├─ "지금 대화하기" → 채팅방
                                └─ "더 둘러보기"   → 홈 피드 (해당 카드 제외)

[바텀 네비 · 데이트 탭] ──→ [캐릭터 프로필 그리드] ──→ 채팅방 / 다시보기
```

## 1. 페이지 (UI 위치)

### 1.1 홈 — 메인 카드 피드

- **상단**
  - 상태바
  - 중앙: Rizz 로고
  - 우측: 코인 잔액 pill
- **본문 (스크롤 영역)**
  - 캐릭터 카드 1장이 본문 영역 풀스크린
  - 카드 내부:
    - **우측 상단**: 태그 칩 행 (예: 미용사 / NTR / BDSM)
    - **좌측 하단**: 캐릭터 이름 + 나이, 대사 한 줄
    - **하단 풀폭**: 주황 CTA "대화 시작하기"
  - **중앙(첫 카드 한정)**: 위로 스와이프 가이드(주황 메시지 박스 + 손 일러스트). 스크롤 시작과 동시에 페이드아웃, 다시 보이지 않음.
- **하단**: 글로벌 바텀 네비 (홈 / 코인 / 데이트 / 채팅 / 마이페이지). 홈 활성.

### 1.2 매칭 성공 모달

- **상단**: 상태바
- **상단 좌측**: 매칭한 캐릭터의 원형 아바타 + 이름
- **본문 중앙**: 캐릭터 풀샷 라운드 사진
- **본문 사진 위 오버레이 (중앙 하단)**: 어두운 카드
  - 체크 아이콘
  - 타이틀: "매칭 성공"
  - 보조 카피: "메시지를 기다리고 있어요!"
- **하단**
  - 1차 CTA(주황): "지금 대화하기"
  - 2차 텍스트 링크(회색): "더 둘러보기"
- 바텀 네비 없음 (모달 성격)

### 1.3 데이트 / 캐릭터 그리드

- **상단**
  - 상태바
  - 중앙: Rizz 로고
  - 우측: 코인 pill
- **상단 메뉴**: 3-탭 (데이트 중 · 캐릭터 프로필 · 다시보기). 활성 탭은 주황 텍스트 + 하단 underline.
- **본문**: 3열 캐릭터 카드 그리드
  - 각 카드:
    - 원형 프로필 사진 (얼굴 중심 크롭)
    - 상태 배지: 원의 **하단 경계를 일부 뚫고 외부로 돌출**
      - 진행 중: 주황 그라데이션
      - 완료: 회색 + 사진 위에 흰색 반투명 오버레이
    - 배지 아래: 캐릭터 이름
- **하단**: 글로벌 바텀 네비. 데이트 활성.

### 1.4 글로벌 요소 (모든 화면 공통)

- **상태바**: 시스템 영역. 화면 배경 톤에 따라 다크/라이트 자동.
- **Rizz 로고**: 탭 시 홈 첫 카드로 복귀.
- **코인 pill**: 탭 시 코인 샵 진입.
- **바텀 네비**: 활성 탭만 주황, 비활성은 회색. 항상 노출(모달 화면 제외).

## 2. 기능 (동작·규칙)

### 2.1 홈 피드

- **카드 풀 로드**
  - 진입 시 추천 캐릭터 N장 서버에서 fetch (개인화 추천 API).
  - 풀 캐시는 세션 단위. 페이지 재진입 시 첫 카드부터.
- **위로 스와이프 / 마우스 드래그업 / 휠 스크롤**
  - 다음 카드로 스냅 전환.
  - 마지막 카드 도달 시: 풀 추가 로드 또는 빈 상태(추후 결정).
- **아래로 스와이프**
  - 직전 카드로 복귀.
- **CTA "대화 시작하기" 탭**
  - 입력: 현재 보이는 카드 캐릭터 ID
  - 동작: 매칭 데이터 생성(서버) → 매칭 성공 모달 진입
  - 실패 케이스
    - 코인/구독 부족 → 코인샵 또는 VIP 페이월 진입
    - 네트워크 오류 → 토스트 + 재시도
- **태그 칩**: 표시만, 인터랙션 없음(v1).

### 2.2 매칭 성공

- 진입 시점에는 이미 매칭 데이터 생성된 상태. 클라이언트는 캐릭터 ID 기준으로 렌더만.
- **"지금 대화하기" 탭**
  - 동작: 해당 캐릭터의 채팅방 진입. 채팅방이 없으면 즉시 생성 후 진입.
- **"더 둘러보기" 탭**
  - 동작: 홈 메인으로 복귀.
  - 규칙: 직전 매칭 카드는 다음 풀 로드 시 제외 또는 우선순위 하향.

### 2.3 데이트 / 캐릭터 그리드

- 진입 시 사용자의 캐릭터 관계 목록 로드 (status: in-progress / done).
- **3-탭 전환**: 동일 캐릭터 풀에 대한 다른 뷰
  - 데이트 중: 현재 진행 중인 관계만
  - 캐릭터 프로필: 사용자가 매칭한(혹은 잠금 해제한) 모든 캐릭터의 프로필 정보
  - 다시보기: 완료/엔딩 회차 회상
- **카드 탭 (진행 중)**
  - 동작: 해당 캐릭터의 채팅방 진입.
- **카드 탭 (완료)**
  - 동작: 다시보기 탭의 해당 회차로 이동, 또는 안내 토스트.
- **빈 상태**
  - 진행 중 0건: "지금 매칭한 캐릭터가 없어요" + 홈으로 가는 버튼
  - 완료 0건: "아직 완료한 캐릭터가 없어요"

### 2.4 글로벌 네비게이션

- **홈 탭**: 홈 첫 카드로. 스크롤 위치 초기화.
- **데이트 탭**: 데이트 화면, 마지막으로 보던 서브탭 유지 (없으면 "데이트 중" 기본).
- **채팅 탭**: 채팅 리스트.
- **코인 / 마이페이지**: 본 스펙 범위 밖.

## 3. 인터랙션 핵심 원칙

| 의도          | 트리거                  | 결과              |
|---------------|-------------------------|-------------------|
| 탐색 (passive)| 위로 스와이프            | 다음 캐릭터 카드  |
| 선택 (active) | CTA "대화 시작하기" 탭   | 매칭 성공 모달    |

→ 두 의도를 의도적으로 분리. 좌우 스와이프(틴더형 binary judgment), O/X 버튼은 채택하지 않음.

## 4. 미정 / 추후 결정

- 카드 풀이 비었을 때 빈 상태 카피·시각
- 코인/VIP 페이월 진입 분기 정확한 조건
- 캐릭터 프로필 탭의 카드 상세 페이지 구조
- 데이트 중 탭의 카드 컴포넌트(현재는 캐릭터 프로필과 동일 — 분리 필요 여부)
- 매칭 직후 첫 메시지: 사용자 발화 vs 캐릭터 발화 시작 정책
